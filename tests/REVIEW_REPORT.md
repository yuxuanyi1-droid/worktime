这里的“功能均有覆盖”表示每个用户功能都有可执行行为证据，不表示每一行代码或每一种外部基础设施故障组合都已自动执行。

## 分功能设计与交互审查

### 认证、SSO 与个人资料

- 密码登录对不存在用户仍执行等成本 bcrypt，失败提示一致，并按 IP + 用户组合限制连续失败，降低账号枚举和暴力破解风险。
- 改密和登出递增 `tokenVersion`，旧 JWT 立即失效；PAT 使用独立撤销模型。
- OIDC state 包含 provider、模式、发起用户、nonce、跳转和过期时间，并由 HMAC 防篡改；外部跳转、伪造 Host、绑定发起者不一致会被拒绝。
- JIT 身份源负责姓名、联系方式和组织归属，本地界面明确只读；补充身份源只同步外部账号展示字段，不覆盖本地档案。
- JIT 建号、组织创建和身份绑定在事务内完成，用户名冲突采用稳定哈希后缀，默认角色缺失时不留下孤儿账号。

### 工时、加班与周报

- 表单以当前系统工时单位约束输入，服务端再次校验项目状态、单位、重复记录、每天上限和提交状态，前端校验不能成为安全边界。
- 修复了编辑工时草稿时可绕过创建校验、改到停用或不存在项目的问题。
- 已审批工时通过废弃旧版本并新建版本修改，审批历史不被覆盖；提交分组序号由专用计数器分配，避免 `MAX+1` 并发冲突。
- 批量提交在事务内完成，通知推迟到提交成功后发送，通知故障不会回滚业务数据。
- 加班统计只聚合已审批数据；周报的自动汇总、草稿编辑和只读状态均有清晰反馈和失败重试。

### 审批

- 待办、历史和我的申请语义分离；申请人可撤回，审批人可同意/驳回，抄送与分享不会改变审批状态。
- 会签/或签使用条件更新和推进乐观锁，避免两名审批人并发处理造成重复推进或永久卡住。
- 同一业务记录只允许一个进行中实例，但保留全部历史实例，驳回/撤回后可以重新提交。
- Redis Streams 为至少一次投递；实例创建具备幂等复用，非法任务进入死信，未确认任务可由其他 Worker 接管。
- 审批流程修改保存版本快照；每种业务类型只允许一个默认流程，默认流程必须可用。

### 项目、组织、角色和权限

- `*:access` 只控制模块入口；查看、创建、编辑、审批、导出等操作由独立权限控制，前端菜单与路由守卫使用同一权限语义。
- 全局权限和范围授权被严格区分。修复了把名称类似 `report:view:all` 的范围授权误判为全局权限、从而扩大报表/项目数据范围的问题。
- 项目管理员、模块 SE、部门和分组范围均由服务端策略计算，不能通过前端隐藏按钮替代鉴权。
- 自定义角色从现有权限目录勾选，不允许随意创建无法落地的权限码；系统角色和最后一名管理员受到保护。
- 分组树响应改为显式安全字段，修复了展开实体关系时可能把用户密码哈希和 `tokenVersion` 带入响应的问题。
- 同部门同父级分组重名、同一用户同 provider 多重绑定、重复生效范围授权均有数据库唯一约束兜底。

### 通知、公告与定时提醒

- 单条已读、全部已读和未读角标使用一致数据语义；已读项从未读弹层移除，完整历史在独立页面展示。
- 公告保持站内消息，同时按全员、部门、含子组的分组或指定用户解析启用账号的 SIAM 工号批量发送 TT；无工号用户只跳过 TT，不影响站内公告。
- 公告编辑不会重复发送 TT，管理端会明确显示发布结果、通道未启用、发送失败或无可用接收人的状态。
- 定时工时提醒校验星期、时间、范围和内容，损坏的历史 JSON 会安全恢复为停用而不是让设置页崩溃；固定使用 Asia/Shanghai。

### 报表与导出

- 个人、组、部门、项目和加班报表的可见范围由服务端返回，前端只渲染可用页签。
- 部门/分组/项目筛选相互联动，切换上级范围时清空失效的下级条件；筛选变化会使在途请求失效，旧响应不能回填新范围。
- 导出权限仍受当前数据范围约束，不能仅凭 `report:export` 导出未授权组织；导出参数和文件生命周期有自动化测试。
- 加班报表统一使用“天”，避免后端 `days` 字段却在界面标成“小时”。

### AI 助手

- 每个用户会话使用独立目录和服务端 sessionId，客户端不能提交文件路径；工具仅开放工时系统只读查询白名单，不提供 bash/read 等通用主机能力。
- 内部查询使用短期 `purpose=agent` JWT，不长期保存 PAT 明文。
- Worker 有生成并发、排队上限、同会话互斥、驻留会话上限、用户历史上限、超时中止和退出清理。
- SSE 处理断流、错误、停止和过期事件；失败发送会恢复输入，切换/删除/重命名会话有确认和状态隔离。
- Markdown 链接限制为相对地址或 HTTP(S)，阻止 `javascript:`；外部图片默认不加载，原始推理内容不直接展示。
- 多 API 部署对 Agent 路由使用签名 Cookie 会话亲和，普通 API 使用最少连接策略。

### 审计、运行时与可运维性

- 审计日志有独立入口和权限，详情 JSON 格式化为可读内容；业务审计失败按既定 best-effort 策略不阻断主操作。
- 安全头、CORS、全局/登录/OIDC 限流和指标令牌均有契约测试。
- API、审批 Worker 和 AI Worker 均覆盖启动失败与优雅退出；Caddy 配置已经二进制校验。
- 测试/恢复/性能脚本已集中到根 `tests/`，生产运行脚本只保留在 `server/scripts/`。
